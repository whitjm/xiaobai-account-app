## 二、产品是什么

**产品名称:** 小白记账

**一句话介绍:** 一款简单易用的桌面记账软件,帮助用户随手记录每一笔人民币花销,并按"大类 / 小类"两级分类管理。

**目标用户:** 想要清晰掌握自己日常开销、但不想被复杂功能困扰的普通人。

**核心价值:** 记账要快、要简单,分类清楚,数据看得明白。

- 必须能运行在 **Windows 电脑**上。
- 必须能运行在 **Mac 电脑**上。

## 四、核心功能需求

软件既能记**支出**,也能记**收入**。

### 4.1 记录一笔账
用户能新增一笔记录,每笔至少包含:
- **类型**(支出 / 收入)
- **金额**(人民币,元,支持小数点后两位,如 12.50)
- **一级分类(大类)**
- **二级分类(小类)**
- **日期**(默认今天,可修改)
- **备注**(可选,比如"和朋友聚餐")

### 4.2 查看记录
- 能看到所有记录的列表(按日期排列)。
- 能看到合计:总支出、总收入、结余。

### 4.3 编辑 / 删除
- 能修改已经记录的某一笔账。
- 能删除某一笔账(删除前给出确认提示,避免误删)。

### 4.4 分类管理
- 采用**两级分类**:一级大类下面挂若干二级小类。
- 支出和收入各有自己的分类体系。
- 软件内置一套默认分类(见第五节),开箱即用。

### 4.5 统计图表(阶段 4)
- 用饼图、折线图等方式直观展示花销/收入情况。

### 4.6 数据导入导出(阶段 5)
- 能把账目导出成文件备份,也能从文件导入。

## 六、技术栈选型

> 状态:**已确定**。技术栈组合为 Electron + React + SQLite。此后技术细节由 AI 按第 1.1 条自行决定。

**桌面框架:** Electron(用网页技术打包成桌面软件)

**框架选择理由:** 用户首次开发,Electron 资料最全、最成熟、遇到问题最容易找到答案;一套代码可同时打包出 Windows 版和 Mac 版;做记账这类小工具绰绰有余,后续想加图表、美化界面都方便。软件体积偏大对记账工具无影响。(2026-07-08 由用户选定)

**界面技术:** React(做界面的工具箱,资料多、生态好,做图表时有大量现成组件)

**界面技术选择理由:** 路线图包含统计图表、导入导出等较丰富的功能,用 React 长期更好维护、做图表更方便。(2026-07-08 由用户选定)

**数据存储方案:** SQLite(数据存在用户本地电脑上的一个数据库文件里)

**存储选择理由:** 数据存本地,安全、不联网、不花钱;SQLite 专为管理数据设计,账目变多也又快又稳,后续做统计/筛选/月度汇总都顺手;备份只需复制一个文件。(2026-07-08 由用户委托 AI 选定)
